사전 준비 ②

섹션 1에 값 입력

파일을 텍스트 에디터로 열어 서비스 URL·Host와 사용할 서비스의 ENABLED 플래그를 입력하세요.
토큰은 여기에 입력하지 않아도 됩니다.

1
URL·Host 입력 + 연결할 서비스에 true 입력
URL은 비워두면 기본값(사내 주소)이 자동 적용됩니다. 사용하지 않을 서비스 플래그는 비워두면 건너뜁니다.
claude-code-onboarding_mac.md
24# Confluence·Jira
25CONFLUENCE_URL=https://wiki.navercorp.com ← URL 입력
26JIRA_URL=https://jira.navercorp.com
27CONFLUENCE_ENABLED=true ← 사용하면 true
28 
29# GitHub
30GITHUB_HOST=비워두면 기본값 자동 적용
31GITHUB_ENABLED=비워두면 건너뜀
32 
33# 네이버 웍스
34NAVER_WORKS_ENABLED=true
35 
36# Figma MCP
37FIGMA_ENABLED=비워두면 건너뜀
38FIGMA_DESKTOP_URL=비워두면 기본값 자동 적용
🔒
토큰은 이 파일에 입력하지 마세요!
자동화 중 Claude가 설정 파일을 생성한 뒤 토큰 입력 위치를 직접 안내합니다.
🚀
코드도 바로 보고 싶다면? (선택)
아래 항목도 함께 입력하면 GitHub 레포를 로컬에 클론해 Claude가 코드를 바로 읽을 수 있어요.
REPO_ORG_PATH=   (예: performance-ad/pa-dsp)
REPO_LOCAL_PATH= (예: ~/dev/pa-dsp)
BRANCH_DEV=      (예: qa2/dev)
BRANCH_REAL=     (예: master)

Figma MCP

Figma MCP 상세 설명

별도 토큰 없이 Claude 계정 OAuth로 자동 인증돼요.
FIGMA_ENABLED=true를 입력하면 자동화 명령 시 Figma도 함께 연결됩니다.

1
자동화 실행 시 두 가지 MCP가 함께 설치됩니다
자동화 명령을 실행하면 Figma MCP도 함께 설치돼요.
figma-remote-mcp — Figma 클라우드 (URL로 파일 접근)
figma-desktop    — Figma 로컬 앱 (현재 열린 파일 직접 접근)
2
figma-desktop 사용 전 — Figma 앱에서 MCP 서버 활성화
처음 한 번만 하면 됩니다.
1. Figma 데스크톱 앱 최신 버전으로 업데이트
2. Figma 디자인 파일 열기
3. 하단 툴바 → Dev Mode 켜기 (단축키: Shift+D)
4. 오른쪽 인스펙트 패널 → MCP server 섹션
5. Enable desktop MCP server 클릭
6. 표시된 URL 확인 (예: http://127.0.0.1:3845/mcp)
💡
🌐 figma-remote-mcp
Figma 앱 안 켜도 됨
Dev Mode 불필요
Figma URL 공유받아 코드 생성
디자인 → 코드 자동화
🖥️ figma-desktop
Figma 앱 켜져 있어야 함
Dev Mode 활성화 필요 (Shift+D)
현재 열린 파일 직접 작업

사전 준비 ③

서비스별 토큰 발급

토큰은 각 서비스가 Claude를 믿고 접근을 허용하는 '열쇠'예요.
지금은 발급해서 메모만 해두세요. 입력은 자동화 중 Claude가 안내합니다.

📝 Confluence (Wiki) 토큰

  1. CONFLUENCE_URL 주소로 접속
  2. 우측 상단 프로필 → Profile
  3. 좌측 메뉴 → Personal Access Tokens
  4. Create token 클릭
  5. 이름 입력 (예: claude-code) → Create
  6. 토큰 복사 → 메모해두기

🎯 JIRA 토큰

  1. JIRA_URL 주소로 접속
  2. 우측 상단 프로필 → Profile
  3. 좌측 메뉴 → Personal Access Tokens
  4. Create token 클릭
  5. 이름 입력 → Create
  6. 토큰 복사 → 메모해두기

🐙 GitHub 토큰

  1. GITHUB_HOST 접속 (사내 네트워크 필요)
  2. 프로필 → Settings
  3. 하단 → Developer settings
  4. Personal access tokens → Tokens (classic)
  5. Generate new token (classic) 클릭
  6. 권한: repo, read:org, read:user 체크
  7. 토큰 복사 → 메모해두기

💬 NAVER WORKS

  1. 파일에 NAVER_WORKS_ENABLED=true 입력
  2. 자동화 실행 중 브라우저 로그인 창이 자동으로 열림
  3. 네이버 SSO 로그인 완료 → Claude에게 알림
⚠️
NAVER WORKS 사용 시 — 자동화 실행 전 필수!
Cursor·Copilot 등 다른 AI 도구에 웍스 MCP가 연결되어 있으면 포트 충돌로 인증이 실패합니다.
웍스 MCP가 연결된 모든 AI 도구를 완전히 종료한 뒤 진행하세요.   macOS: Cmd+Q  |  Windows: Alt+F4
🎨
Figma는 토큰 발급이 필요 없어요!
자동화 완료 후 첫 호출 시 Claude 계정으로 OAuth 인증이 자동으로 진행됩니다.

자동화 실행 ①

Claude에게 자동화 명령

준비가 끝났으면 Claude Code를 열고 명령 하나만 입력하세요.
이후 설치·설정은 Claude가 자동으로 처리합니다.

1
터미널에서 Claude Code 실행
설정 파일이 있는 폴더에서 터미널을 열고 claude를 입력하세요.
Terminal
user@mac ~/Downloads $ claude
╭──────────────────────────────────────────────╮
│ Claude Code · claude.ai/code │
╰──────────────────────────────────────────────╯
2
아래 문장을 그대로 입력
Claude Code 프롬프트에 아래 문장을 입력하세요. 파일 경로를 함께 알려주면 더 정확해요.
Claude Code
> 이 파일 읽고 MCP 세팅 자동화해줘
또는 경로를 직접 지정:
> ~/Downloads/claude-code-onboarding_mac.md 읽고 MCP 세팅 자동화해줘
✅
Claude가 파일을 읽고 설치할 서비스 목록을 확인한 뒤 자동으로 진행합니다.
중간에 몇 가지 직접 입력이 필요한 단계가 있어요 — 다음 슬라이드에서 안내합니다.
7 / 10

자동화 실행 ②

토큰 입력 & 웍스 로그인

자동화 중 두 번 사용자 개입이 필요합니다.
Claude가 안내할 때 아래 순서대로 진행해주세요.

1
settings.json에 토큰 직접 입력
Claude가 ~/.claude/settings.json을 생성하고 토큰 입력 위치를 안내합니다.
텍스트 에디터로 파일을 열어 플레이스홀더를 발급한 토큰으로 교체하세요.
수정 전
settings.json
3 "env": {
4 "CONFLUENCE_PERSONAL_TOKEN":
5   "YOUR_CONFLUENCE_TOKEN"
6 }
→
수정 후
settings.json
3 "env": {
4 "CONFLUENCE_PERSONAL_TOKEN":
5   "MTQ1MTIx..." ✅
6 }
입력 완료 후 Claude에게 "토큰 입력 완료했어"라고 알려주세요.
2
[NAVER WORKS 연결 시] 브라우저 로그인
자동화 중 브라우저가 자동으로 열리면 네이버 SSO 계정으로 로그인하세요.
브라우저
🔒 https://kr0-prismgw.io.naver.com/oauth/authorize?...
NAVER
네이버 통합 SSO 로그인
아이디
비밀번호
로그인 완료 후 Claude에게 "웍스 로그인 완료했어"라고 알려주세요.

자동화 실행 ③

완료 보고 확인 & 재시작

거의 다 왔어요! 마지막 두 단계만 남았습니다.

1
자동화 완료 보고 확인
Claude가 아래와 같은 완료 보고를 보내면 설치가 끝난 거예요.
✅ mcp-atlassian (Confluence + Jira) 설치 완료
⏭️ mcp-oss (GitHub) 건너뜀
✅ naver-works (네이버 웍스) 설치 완료
✅ figma-remote-mcp — settings.json 등록 완료 (재시작 후 별도 확인)
✅ figma-desktop — settings.json 등록 완료 (재시작 후 별도 확인)
2
Claude Code 재시작
설정이 적용되려면 Claude Code를 완전히 종료 후 재실행해야 합니다.
🍎 macOS: Cmd+Q 후 재실행
🪟 Windows: Alt+F4 또는 작업 표시줄 우클릭 → 창 닫기 후 재실행
3
[Figma 사용 시] 재시작 후 새 대화에서 인증
재시작 후 새 대화를 열고 아래 문장을 입력하면 Figma OAuth 인증 및 연결 검증이 자동으로 진행됩니다.
Claude Code — 새 대화
> figma-remote-mcp를 호출하여 인증 진행 및 연결 정상 여부 파악
브라우저에 Figma 로그인 팝업이 열리면 로그인하면 자동 완료됩니다.
⚠️
재부팅 후 유의사항: Podman 재시작 필수
컴퓨터를 재부팅하면 Podman 가상 머신이 중지됩니다. Claude Code 실행 전에 반드시 Podman을 먼저 시작하세요.

방법 1. 명령어로 실행
podman machine start
방법 2. Podman Desktop 앱으로 실행
🍎 macOS: Finder/Launchpad에서 Podman Desktop 실행 → Running 상태 확인
🪟 Windows: 시작 메뉴에서 Podman Desktop 실행 → Running 상태 확인
